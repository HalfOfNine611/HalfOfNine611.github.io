RabbitMQ入门（3）-Java实现Fanout，Direct，Topics模式
发表于2025-10-07
1.Exchanges简述RabbitMQ 消息传递模型的核心思想是：生产者生产的消息从不会直接发送到队列。实际上，通常生产者甚至都不知道这些消息传递传递到了哪些队列中。 相反，生产者只能将消息发送到交换机(exchange)，交换机工作的内容非常简单，一方面它接收来自生产者的消息，另一方面将它们推入队列。交换机必须确切知道如何处理收到的消息。是应该把这些消息放到特定队列还是说把他们到许多队列中还是说应该丢弃它们。这就的由交换机的类型来决定。 Exchanges 的类型总共有四种：直接(direct)、主题(topic)、标题(headers)、扇出(fanout) 生产者发消息都是发给交换机，由交换机对消息进行路由到队列，那么交换机这么知道将消息发给哪个队列呢？这就是 routing-key 要做的事，它是 exchange 和 queue 之间的桥梁，告诉我们 exchange 和那个队列进行了绑定关系。比如说下面这张图告诉我们的就是 X 与 Q1 和 Q2 进行了绑定。 queueBind方法中的第三个参数就是设定routing-key的值，用于设定交换机和队列的绑定关系...

RabbitMQ入门（1）--利用Docker进行安装
发表于2025-09-29
开始之前默认有Linux虚拟机，SSH客户端工具，已安装Docker，有rabbitmq.tar包 1.将tar包拖入虚拟机tmp文件夹下（我使用的是Xterminal客户端工具） 2.进入tmp文件夹查看，rabbitmq.tar已在其中 使用命令docker load -i rabbitmq.tar来添加到docker的镜像中,添加成功后使用命令docker images查看镜像 3.使用命令创建容器，user，pass部分自定义 docker run -e RABBITMQ_DEFAULT_USER=你的名称 -e RABBITMQ_DEFAULT_PASS=你的密码 --name mq --hostname mq1 -p 15672:15672 -p 5672:5672 -d rabbitmq:3.8-management 4.使用虚拟机地址:15672访问rabbitmq，使用刚才自定义的user，pass登录即可
